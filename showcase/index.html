Technical JSON

How it works

Deterministic keyword/pattern rules — explainable, offline, not “AI understanding.”

What it detects

Crisis language, distress signals, toxicity/harassment cues, night-support cues, and optional geofence events.

When a signal appears

The UI shows severity, matched indicators, recommended local actions, and optional SQLite alert persistence.

Cloud required?

No. Default operation is offline-first on your machine. No analytics or telemetry by default.

What is stored

Alert summaries in local SQLite. Raw message text is not retained unless you explicitly enable it.

Evidence

Verified capabilities — not marketing claims.

  • Evidence index
  • Detector evaluation method
  • GET /ready — storage readiness
  • GET /status — runtime honesty JSON

Privacy

Local-first defaults. See PRIVACY.md.

  • No telemetry by default
  • No third-party fonts or trackers in this UI
  • Raw message retention off by default
  • Authenticated users can delete their alerts
  • Full reset: stop service and delete the local SQLite file

Safety boundaries

Say what this is. Say what it is not.

This is

  • A supportive technical demonstration
  • A deterministic safety-signal scanner
  • Informational crisis-resource routing
  • Local-first infrastructure for sensitive demos

This is not

  • Therapy or a clinician
  • A medical device or diagnostic tool
  • An emergency service or hotline
  • A replacement for parents, clinicians, 988, or 911

If someone may be in immediate danger, call or text 988 (US) or contact emergency services.

Deployment

One command for a local demonstration.

docker compose up --build
curl http://127.0.0.1:8000/ready
open http://127.0.0.1:8000/

Local Python bind defaults to 127.0.0.1. LAN exposure requires explicit configuration. See DEPLOYMENT.md.
