# Roadmap

## Near-term priorities

### 1. Repository consolidation

- remove duplicate root-level safety module
- simplify root structure
- unify Python dependency path
- clean test organization

### 2. Documentation polish

- finalize architecture, safety, and demo docs
- add screenshots and visual assets
- make setup paths clearer for new visitors

### 3. Demo improvements

- unify Node and backend demonstration flow
- add example request and response payloads
- create a cleaner public-facing walkthrough

---

## Product-facing priorities

### 4. Packaging

- prepare a cleaner demo release
- add versioned releases
- improve Docker documentation
- define deployment profiles

### 5. Local-first deployment

- improve mini-PC and edge deployment notes
- explore Raspberry Pi-friendly path
- document local journaling storage options

### 6. Integrations

- define cleaner interface between empathy layer and backend
- support pluggable response engines
- support use as a modular product component

---

## Business-facing priorities

### 7. Pilot readiness

- shape repo into a strong flagship showcase
- define nonprofit pilot package
- define white-label deployment direction
- add clearer product brief and buyer-facing language

### 8. Trust and safety maturity

- improve escalation documentation
- tighten alert behavior and privacy language
- document human review boundaries more explicitly

---

## Longer-term direction

Possible future directions include:

- white-label empathy and safety layer
- family-support edge deployment package
- nonprofit or school pilot bundle
- clinician-adjacent journaling assistant stack
- private youth-support AI appliance concept

---

## Success criteria

This roadmap is working if the project becomes:

- easier to understand in under 60 seconds
- stronger as a recruiter-facing flagship repo
- stronger as a founder-facing product seed
- stronger as a trust-centered, privacy-first AI example

---

## Summary

The goal is not to make the repo bigger.

The goal is to make it sharper, clearer, and more credible.
